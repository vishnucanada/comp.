The problem

You're already calling LLMs. You don't yet have a record of why.

Every team gating a Claude or GPT call cobbles together a mix of system prompts, keyword regexes, and ad-hoc moderation calls. There is no single record of what was blocked, by which rule, for which user.

How it works

One gate, one log, one policy file

comp. is a small Python library. Wrap your existing LLM call with PolicyGate and every request goes through the same pipeline: policy → guard → decision → log.

1
Write a policy file
A short DENY/ALLOW DSL with keyword matchers and optional regexes. Keep it in your repo so changes go through the same review process as code.
policy.txt
2
Pick a guard backend
KeywordGuard for the cheap default. OpenAIModerationGuard for free, fast, managed-taxonomy detection. LlamaGuardGuard for self-hosted deployments where outside API calls aren't an option. Compose multiple guards if you want.
make_guard(policy, backend="openai")
3
Wrap your existing LLM call
Pass any callable to gate.complete(). If the policy denies the prompt the callable is never invoked; otherwise it runs unchanged. No request reaches your model without a recorded decision.
gate.complete(prompt, fn=lambda m: claude(m))
4
Review the audit log
Every decision is appended as one JSONL line with a hashed prompt, the user role, the categories that fired, and which guard returned the verdict. Aggregate from the dashboard or pipe to your SIEM.
audit/gate.jsonl
Features

Operational tools that read like normal Python

No special runtime, no model retraining. Bring your own LLM.

📝
Policy DSL in plain text
DENY / ALLOW rules with keyword matchers and optional regexes. Lives in your repo. Diffs cleanly. No magic.
🔌
Bring your own LLM
gate.complete() takes any callable. Works with Claude, GPT, Ollama, local Hugging Face models, or whatever you already have.
🛡️
Pluggable content guards
KeywordGuard (cheap), OpenAIModerationGuard (free, fast), LlamaGuardGuard (self-hosted), or a CompositeGuard over several at once.
🔐
IAM-style roles
YAML-defined roles, full-privilege bypass for admins, tool allowlists per role. Unknown role names never silently inherit defaults.
📜
Append-only audit log
Every decision lands in a JSONL log: timestamp, hashed prompt, role, verdict, fired categories, guard backend. Pipe straight to your SIEM.
⚡
Streaming-friendly
Server-Sent Events from the dashboard chat with the policy decision in the first chunk, then the model's tokens.
📊
Compliance reports
comp report renders Markdown / JSON summaries from the audit log: deny rate, top categories, requests per role, decisions per backend.
🧰
Zero runtime dependencies
The core library is stdlib only. Each guard backend is an opt-in extra: comp[openai], comp[llamaguard], comp[dashboard].
Guard backends

Pick a detection engine that fits your environment

The keyword guard is fast and deterministic but trivially bypassed by paraphrase. For real-world deployments, plug in a managed or self-hosted classifier.

KeywordGuard
stdlib only · keyword + regex + paraphrase map
OpenAI Moderation
free · OPENAI_API_KEY required · low latency
Llama Guard 3
self-hosted · transformers + 16 GB · gated repo
CompositeGuard
OR over several backends · merged categories
Custom
subclass Guard, return a GuardResult
Developer experience

Wrap your existing call

No proxy, no new endpoint. The gate is a Python object — pass your model call as a callable.

integration.py — gate any LLM 
from anthropic import Anthropic
from src import Policy, PolicyGate, IAMConfig, make_guard

policy = Policy.from_file("policies/hr.txt")
guard  = make_guard(policy, backend="openai")        # or "keyword", "llamaguard"
iam    = IAMConfig.from_yaml("iam.yaml")
gate   = PolicyGate(policy, guard=guard, iam=iam,
                    audit_log_path="audit/gate.jsonl")

claude = Anthropic()

# If the policy denies the prompt, claude is never called.
response, decision = gate.complete(
    message=user_input,
    fn=lambda m: claude.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=512,
        messages=[{"role": "user", "content": m}],
    ).content[0].text,
    user_role="analyst",
)

print(decision)
