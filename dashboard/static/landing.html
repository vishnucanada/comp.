How it works

Compliance baked into the weights

comp. replaces every Linear layer in your model with a rank-factorized equivalent. At low privilege only the first g rank-1 components are active — the computation path for denied information simply doesn't exist.

1
Define your compliance policy in plain English
Open the dashboard and paste your compliance requirements exactly as you'd write them for legal — or load one of our pre-built HIPAA, GDPR, PCI-DSS, SOC 2, CCPA, or financial-services policies with one click.
60-second setup
2
We fine-tune your model's weights — not a wrapper
Algorithm 1 adjusts the B matrices in NLPN layers so that at low privilege the model genuinely can't compute denied answers, while full-privilege behaviour is provably preserved. Training takes 3–8 minutes on an M-series Mac.
No capability regression
3
Every request is privilege-allocated in real time
The policy compiler checks each prompt against your rules. A DENY match drops the model to low privilege — the denied capability is structurally unreachable for that generation. No post-processing, no second model, no latency hit on the ALLOW path.
g = 1% of rmax on DENY
4
Compliance auditors verify the math, not the logs
Because Im(W(g)) ⊆ Im(W(g+1)) is provable from the weight matrices alone, your auditor doesn't need to run the model to confirm compliance. GDPR tiered enforcement and tamper-evident audit logs are built in.
Verifiable without inference

Developer experience

One line to switch from OpenAI

Change the base URL and add a policy header. Every existing OpenAI call now runs through NLPN enforcement.

integration.py — OpenAI drop-in 
# Before: direct OpenAI
from openai import OpenAI
client = OpenAI()

# After: comp. enforcement (one line change)
client = OpenAI(
    base_url="http://localhost:8000/v1",
    api_key="your-comp-api-key",
    default_headers={"policy": "hipaa"},
)

# Same call — NLPN enforcement is transparent
response = client.chat.completions.create(
    model="comp-enforced",
    messages=[{"role": "user", "content": prompt}],
)

# Bonus: response includes enforcement decision
print(response.model_extra["x_comp_policy"])
# {'decision': 'DENY', 'violations': ['patient identifiers'], 'policy': 'hipaa'}
